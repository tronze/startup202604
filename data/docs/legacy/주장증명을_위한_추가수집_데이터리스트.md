# 주장 증명을 위한 추가 수집 데이터 리스트 (확장판)

## 1. 문서 목적

본 문서는 `스타트업 아이디어/주장과 근거`의 주장(특히 B/C 수준)을 **실증 가능한 형태**로 바꾸기 위해 필요한 데이터 수집 항목을 전체적으로 정리한 실행 문서다.

핵심 목표:

- 누락 없이 증명 데이터 풀 구성
- 주장별 필수 데이터와 보강 데이터를 구분
- 수집 우선순위와 운영 절차를 바로 실행 가능하도록 정의

## 2. 증명 프레임 (주장 ↔ 증거)

| 주장 | 증명하려는 질문 | 최소 증거 조건 |
| --- | --- | --- |
| 주장 3 (손실 시각화 효용) | 실제로 얼마를 못 팔았고, 손실이 얼마인가? | 출력제어 실측(MW/MWh) + 정산단가 + 계약단가 |
| 주장 4 (대체 수익화) | ESS/PPA/운영전략으로 손실을 줄일 수 있는가? | 반사실 시뮬레이션 가능 데이터(운영 로그+시장가격+제약) |
| 주장 5 (지역 제약 인텔리전스) | 어느 지역/변전소/선로가 구조적으로 막히는가? | 계통 혼잡 실측 + 접속 이력 + 보강 계획 |
| 주장 6 (정책형 밸런싱) | 수요를 어디로 유도하면 계통 효율이 개선되는가? | 지역 공급/수요/요금/인센티브 + 입지 이동 결과 |
| 주장 7 (플랫폼 ROI) | 제품이 실제 의사결정과 수익 개선을 만들었는가? | 고객 행동 로그 + 전후 성과 KPI |

## 4. 우선순위 (90일 실행)

### 1순위: 반드시 확보

- 출력제어 지시 로그
- 발전소별 실발전량 + 정산서
- SMP/REC 시계열
- ESS 운영 로그(최소 1개 파트너)

### 2순위: 제품 확장 핵심

- 변전소/선로 혼잡 및 역송전
- 접속 대기/반려/지연 이력
- 송배전 보강 계획

### 3순위: 정책/입지 고도화

- 산업단지/공장 부하
- 지역 인센티브 및 입지 이동 결과
- 제품 사용 로그 기반 ROI 데이터

## 6. 누락 방지 체크리스트

- [ ] 실발전량과 잠재발전량이 분리 수집되는가
- [ ] 출력제어 지시 데이터에 대상 자산 식별자가 있는가
- [ ] 가격 데이터(SMP/REC/계약단가)가 같은 시간축으로 정렬 가능한가
- [ ] 계통 제약 데이터가 변전소/선로 단위로 매핑 가능한가
- [ ] 접속 이력이 좌표 또는 행정코드로 위치화 가능한가
- [ ] 정책 데이터가 시행일 기준으로 시계열 비교 가능한가
- [ ] 고객 KPI가 도입 전/후 비교 가능하게 저장되는가

## 7. 필수 메타데이터 표준

- 수집출처(기관/URL/담당자)
- 데이터 기준시점(작성일/갱신주기)
- 시간해상도(5분/15분/시간/일/월)
- 공간해상도(전국/권역/변전소/선로/사업장)
- 단위 및 집계 규칙
- 결측/보정 규칙
- 이용권한(내부/외부공유/재배포 제한)
