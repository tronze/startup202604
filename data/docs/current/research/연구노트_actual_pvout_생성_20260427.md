# 연구노트: 지역별 actual PVOUT 생성

작성일: 2026-04-27  
작성 목적: KPX 공개데이터를 이용해 `지역별 실제 단위용량당 발전량`을 만들고, 이를 Solargis `PVOUT`와 비교 가능한 단위로 정리한다.

## 1. 문제의식

지금까지 보던 `Solargis PVOUT`은 장기평균 기반의 잠재량 값이다.  
하지만 실제 사업 판단에서는 “그 지역 태양광이 실제로 얼마나 생산되고 있는가”를 같은 단위로 보고 싶다.

단순 `총발전량(MWh)`만으로는 비교가 안 된다. 지역마다 깔린 설비용량 `MW`가 다르기 때문이다.  
따라서 아래 질문을 풀어야 했다.

1. 지역별 태양광 발전량이 있는가
2. 같은 지역, 같은 기간의 태양광 설비용량이 있는가
3. 두 값을 나눠 `kWh/kWp/day` 형태로 바꿀 수 있는가

## 2. 결론

가능하다.  
KPX 공개자료만으로 `지역별 actual PVOUT`를 만들 수 있다.

계산식은 아래와 같다.

```text
actual_pvout_kwh_per_kwp_day
= 지역 월간 발전량(MWh) / 지역 월간 설비용량(MW) / 그달 일수
```

`MWh / MW / day`는 수치상 `kWh / kW / day`와 동일하므로, Solargis `PVOUT`와 직접 비교 가능한 단위다.

## 3. 사용 데이터

### 3-1. 지역별 시간별 태양광 발전량

- 출처: KPX 지역별 시간별 태양광 및 풍력 발전량 파일
- 로컬 파일: `data/raw/kpx_generation_download`
- 단위: `MWh`
- 주요 필드:
  - `거래일`
  - `거래시간`
  - `지역`
  - `연료원`
  - `전력거래량(MWh)`

이번 계산에서는 `연료원 = 태양광`만 사용했다.

### 3-2. 지역별 태양광 설비용량

- 출처: KPX 신재생연계ESS 설비용량 자료 ZIP
- 로컬 파일: `data/raw/kpx_capacity_download`
- 내부 CSV:
  - `한국전력거래소_신재생연계ESS 설비용량자료_20240630.csv`
  - `한국전력거래소_신재생연계ESS 설비용량자료_202507.csv`
- 단위: `MW`

CSV 구조는 시점에 따라 조금 다르다.

- 구버전 컬럼: `태양광 전체 설비용량`
- 신버전 컬럼: `태양광`

둘 다 같은 의미의 태양광 지역별 설비용량으로 해석해 사용했다.

## 4. 처리 과정

### 4-1. 발전량 월집계

시간별 태양광 발전량을 `연도-월-지역` 기준으로 합산했다.

### 4-2. 설비용량 월별 정리

ZIP 내부 두 CSV를 모두 읽고 `연도-월-지역` 기준으로 정리했다.  
겹치는 기간은 더 최신 파일 값을 우선 사용했다.

### 4-3. 지역명 정규화

설비용량 파일과 발전량 파일은 지역명이 완전히 같지 않았다.

예:
- `서울특별시` -> `서울시`
- `부산광역시` -> `부산시`
- `세종특별자치시` -> `세종시`

이 매핑을 정규화해서 조인했다.

참고로 발전량 파일에는 `육지`라는 값도 있었는데, 이 값은 `풍력`에만 있었고 태양광 계산에는 영향이 없었다.

### 4-4. actual PVOUT 계산

월별 지역행마다 아래 값을 계산했다.

- `solar_generation_mwh`
- `solar_capacity_mw`
- `days_in_month`
- `actual_pvout_kwh_per_kwp_day`
- `annualized_kwh_per_kwp_year`

## 5. 산출물

- 월별 테이블: `data/processed/actual_pvout_region_monthly.csv`
- 연간 요약: `data/processed/actual_pvout_region_annual.csv`
- 생성 스크립트: `scripts/build_actual_pvout_region_monthly.py`

또한 SQLite에도 아래 테이블로 적재했다.

- `actual_pvout_region_monthly`
- `actual_pvout_region_annual`

## 6. 현재 확보 범위

실제 매칭 가능한 기간은 `2025-01`부터 `2025-05`까지다.

이유는:
- 발전량 파일은 `2025-01-01 ~ 2025-12-31`
- 설비용량 파일은 최신 CSV 기준 `2025-05`까지 포함

따라서 지금 만든 연간 요약은 엄밀히 말하면 `2025년 1~5월 평균`이다.  
완전한 2025 연간 실적표로 해석하면 안 된다.

## 7. 첫 결과

`2025-01 ~ 2025-05` 기준 지역별 actual PVOUT 상위권은 아래와 같다.

- 부산시: `3.813 kWh/kWp/day`
- 울산시: `3.755 kWh/kWp/day`
- 대구시: `3.660 kWh/kWp/day`
- 서울시: `3.585 kWh/kWp/day`
- 전라북도: `3.581 kWh/kWp/day`

즉 같은 KPX 시장참여 설비 기준으로 보면, 최소한 2025년 초반 실적에서는 지역별 차이가 분명히 나타난다.

## 8. 해석상의 주의점

이 값은 유용하지만, 그대로 “한국 전체 태양광의 절대 효율”로 읽으면 안 된다.

이유는 아래와 같다.

1. 발전량과 설비용량 모두 `KPX 전력시장 참여 설비` 기준이다.
2. 한전 직접거래(PPA), 자가용, 비시장 설비는 빠질 수 있다.
3. 따라서 이 값은 “시장참여 설비 내부 비교용 actual PVOUT”으로 해석하는 것이 정확하다.
4. 또한 현재는 `1~5월`까지만 있어 계절 편향이 남아 있다.

## 9. 이번 발견의 의미

이번 결과로 확인한 것은 두 가지다.

1. `지역별 실제 발전성과`를 `PVOUT`와 같은 단위로 바꾸는 것은 가능하다.
2. 따라서 앞으로는 `Solargis 잠재량`과 `KPX 실제 성과`를 한 표에서 비교할 수 있다.

이 비교는 중요하다.  
왜냐하면 Solargis는 “햇빛 잠재량”이고, 이번 산출물은 “실제 운영 성과”이기 때문이다.

즉 앞으로는 다음과 같은 질문이 가능해진다.

- 자원은 좋은데 실제 성과가 낮은 지역은 어디인가
- 실제 성과가 잠재량보다 유난히 좋은 지역은 어디인가
- 계통, 운영, 출력제어 영향이 의심되는 지역은 어디인가

## 10. 다음 단계

우선순위는 아래와 같다.

1. Solargis raster에서 지역별 평균 `PVOUT`를 추출한다.
2. `actual_pvout_region_monthly`와 붙여서 `잠재량 대비 실제 성과 비율`을 만든다.
3. KPX 출력제어 공지, SMP, 수요/예비율 데이터를 추가 레이어로 결합한다.
4. 결국 `자원`, `실적`, `가격`, `리스크`를 분리한 지역 점수체계로 확장한다.

## 11. 한 줄 결론

이제 우리는 `총발전량`만 보는 단계가 아니라, `지역별 실제 PVOUT`를 계산해서 Solargis와 직접 비교할 수 있는 단계로 넘어왔다.
