# 태양광 입지 및 수익추정 데이터 상세도

기준일: 2026-04-27

## 결론

우리가 생각하는 모델은 가능하다.

목표 모델:

> 어느 지역/격자/필지에 일조량이 얼마나 생기는지 보고, 그 위치에 태양광 발전기를 설치했을 때 연간 발전량과 예상 수익을 추정한다.

다만 데이터 상세도에 따라 결과 수준이 달라진다.

- 시도/시군구 수준: 빠르게 비교 가능
- 격자 수준: 전국 후보지 ranking 가능
- 필지/건물 수준: 사업 검토에 가까운 수준
- 실제 투자 판단 수준: 계통 접속, 토지, 인허가, 출력제어, CAPEX/OPEX까지 필요

## 1. 받을 수 있는 데이터 상세도

### A. 현재 보유 데이터

현재 repository 안의 기상 데이터:

- 파일: `data/processed/weather_daily.csv`
- 지역: 서울, 인천, 부산, 대구 4개 도시
- 단위: 일별
- 필드: 평균/최고/최저기온, 강수량, 일사량, 풍속

가능한 것:

- 4개 도시 간 일사량 비교
- 계절별 일사량 패턴
- 수요/기상/태양광 이용률 간 rough comparison

불가능한 것:

- 전국 모든 지역 상세 ranking
- 시군구/필지 단위 적합도
- 발전소별 수익 추정

### B. 기상청 ASOS 관측자료

출처:

- 기상청 ASOS 시간자료 API: https://www.data.go.kr/data/15057210/openapi.do
- 기상청 ASOS 일자료 API: https://www.data.go.kr/data/15059093/openapi.do

상세도:

- 공간: ASOS 관측소 지점 단위
- 시간: 시간별 또는 일별
- 기간: 1904년 4월부터 현재까지, 단 지점별 상이
- 제공: 기온, 강수량, 기압, 습도, 일사, 일조 등
- 시간자료에는 `일사(icsr)`, `일조(ss)` 필드가 있다.

가능한 것:

- 관측소별 장기 일사량 ranking
- 시간대별 일사/일조 패턴 분석
- 연도별 변동성, 장마/폭염/계절성 분석
- 실제 발전량 예측 모델의 기상 feature 생성

한계:

- 관측소 위치 기준이다. 산지, 해안, 도시 내 위치 차이를 모두 반영하지 못한다.
- 관측소가 없는 지역은 보간해야 한다.
- 지형 음영, 건물 그림자, 경사/방위는 별도 GIS가 필요하다.

### C. Global Solar Atlas / Solargis GIS 데이터

출처:

- Solargis Korea download: https://solargis.com/maps-and-gis-data/download/republic-of-korea
- Global Solar Atlas / World Bank dataset: https://datacatalog.worldbank.org/search/dataset/0038641/world-photovoltaic-power-potential-pvout-gis-data-global-solar-atlas

받을 수 있는 layer:

- `GHI`: Global Horizontal Irradiation, 수평면 전일사량
- `DNI`: Direct Normal Irradiation, 직달일사량
- `DIF`: Diffuse Horizontal Irradiation, 산란일사량
- `GTI`: Global Tilted Irradiation, 최적 경사면 일사량
- `PVOUT`: 태양광 발전 잠재량, kWh/kWp
- `TEMP`: 온도
- `OPTA`: 최적 경사각

상세도:

- GHI, DIF, GTI, DNI: 약 250m 격자
- PVOUT, TEMP: 약 1km 격자
- OPTA: 약 4km 격자
- 형식: GeoTIFF
- 시간 표현: 장기 평균 월별/연간 totals 또는 daily totals

가능한 것:

- 전국 격자별 태양광 발전 잠재량 지도
- 지역/시군구별 평균 PVOUT ranking
- 특정 좌표의 예상 kWh/kWp/year 산출
- 후보 부지 주변 격자 비교
- QGIS 또는 Python raster 처리로 지도 분석

한계:

- 장기 평균 자원지도라서 특정 연도의 실제 날씨 수익은 별도 보정 필요
- 계통 접속, 출력제어, 토지 규제, 인허가, 공사비는 반영하지 않는다.

### D. 필지/건물 단위 분석에 필요한 추가 GIS

필지/건물 단위로 가려면 아래가 필요하다.

| 데이터 | 목적 |
| --- | --- |
| DEM/DSM | 경사, 방위, 산지 음영 계산 |
| 건물 footprint/높이 | 지붕형 태양광 가능 면적, 그림자 |
| 연속지적도/토지이용 | 필지 경계, 지목, 이용 제한 |
| 도로/전주/배전선/변전소 위치 | 공사비 및 접속 가능성 추정 |
| 개발행위/산지/농지 규제 | 인허가 가능성 판단 |
| 한전 계통 접속 가능 용량 | 실제 사업 가능성 판단 |
| 출력제어/혼잡 이력 | 발전량 손실 risk 반영 |

이 단계부터는 단순 기후 분석이 아니라 “입지 사업성 모델”이 된다.

## 2. 수익추정 모델 구조

### 1단계: 발전량 추정

가장 단순한 식:

```text
연간발전량(kWh/year)
= 설비용량(kWp) * PVOUT(kWh/kWp/year)
```

예:

```text
100kW 설비
PVOUT 1,350 kWh/kWp/year 지역
=> 연간발전량 약 135,000 kWh/year
```

PVOUT 대신 GHI만 있을 때:

```text
연간발전량
= 설비용량 * 연간일사량 * 성능계수(PR) * 보정계수
```

하지만 실무에서는 PVOUT layer를 쓰는 편이 더 빠르고 안정적이다. PVOUT은 이미 태양광 발전 잠재량으로 계산된 지표이기 때문이다.

### 2단계: 매출 추정

발전소형 매출:

```text
연간매출
= 연간발전량 * 판매단가
```

판매단가는 사업모델에 따라 다르다.

| 모델 | 단가 구성 |
| --- | --- |
| 전력시장 판매 | SMP + REC |
| 고정가격계약 | 장기계약 단가 |
| PPA | 계약단가 |
| 자가소비 | 회피 전기요금 단가 |
| ESS/DR 결합 | 전력판매 + 요금절감 + 보조서비스/DR 수익 |

예시 계산:

```text
100kW
연간발전량 135,000 kWh
가정 판매단가 140원/kWh
=> 연매출 18,900,000원
```

주의:

- 140원/kWh는 예시 단가다.
- 실제 수익 모델은 해당 시점의 SMP, REC, 계약단가, 계통한계가격, REC 가중치, 정산 구조를 넣어야 한다.

### 3단계: 비용 및 손실 반영

```text
영업현금흐름
= 연간매출
- O&M
- 보험/임대료/관리비
- 계통접속/망 이용 관련 비용
- 출력제어 손실
- 성능저하 손실
```

반드시 반영해야 할 손실:

| 손실 | 설명 |
| --- | --- |
| 시스템 손실 | 인버터, 배선, 오염, mismatch |
| 온도 손실 | 고온 지역에서 패널 효율 하락 |
| 음영 손실 | 산/건물/전주/수목 그림자 |
| 가동중단 | 고장, 점검 |
| 출력제어 | 계통 지시로 발전해도 팔지 못하는 손실 |
| 노후화 | 연 0.3-0.7% 수준의 degradation 가정 필요 |

### 4단계: 투자성 지표

```text
CAPEX = 설비용량(kW) * 설치단가(원/kW)
단순회수기간 = CAPEX / 연간영업현금흐름
NPV = 20년 현금흐름 할인합 - CAPEX
IRR = NPV가 0이 되는 할인율
```

## 3. 모델을 만들 때의 현실적인 상세도

### 빠른 버전: 시군구 ranking

입력:

- Global Solar Atlas PVOUT 1km 격자
- 행정구역 경계
- 지역별 SMP/REC는 전국 동일 또는 권역 보정

출력:

- 시군구별 평균 PVOUT
- 100kW/500kW/1MW 기준 예상 연간 발전량
- 예시 단가별 연매출

장점:

- 빠르게 만들 수 있다.
- 투자자/내부 검토용 map으로 좋다.

한계:

- 실제 필지 사업성은 아니다.

### 중간 버전: 격자별 후보지 ranking

입력:

- PVOUT 1km
- GHI/GTI 250m
- DEM 경사/방위
- 토지이용/지목
- 도로/전력망 근접도

출력:

- 격자별 태양광 잠재 발전량
- 토지/경사/계통 가중 score
- 후보지 top list

장점:

- 실제 사업 후보지 탐색에 가까워진다.

한계:

- 계통 접속 가능 용량이 없으면 “좋아 보이지만 못 붙이는 땅”이 섞인다.

### 고급 버전: 필지별 수익성 모델

입력:

- 필지 경계
- 건물/지형 음영
- 배전선/변전소/접속 여유
- 출력제어 risk
- CAPEX/OPEX
- SMP/REC/PPA scenario

출력:

- 필지별 예상 발전량
- 연매출, EBITDA, 회수기간, IRR
- 출력제어 민감도
- 접속 리스크 score

장점:

- 실제 사업개발/영업용으로 쓸 수 있다.

한계:

- 한전 계통 데이터와 인허가 데이터 확보가 병목이다.

## 4. 우리가 먼저 만들 수 있는 MVP

MVP는 아래 정도가 적당하다.

입력:

- Solargis/Global Solar Atlas `PVOUT`, `GHI`
- 행정구역 경계
- 현재 보유 `renewable_utilization_monthly.csv`
- 현재 보유 `regional_energy_usage_2024.csv`
- 출력제어 관련 KPX 공지

출력:

- 시군구/시도별 태양광 발전 잠재량 ranking
- 100kW, 500kW, 1MW 설치 시 예상 연간발전량
- 단가 scenario별 예상 매출
- 출력제어/계통 risk는 별도 warning score

첫 화면에서 보여줄 수 있는 문장:

> 이 지역은 태양광 자원 기준으로 연간 약 X kWh/kWp 발전 잠재량이 있고, 500kW 설치 시 연간 약 Y MWh 생산이 예상된다. 단가 A/B/C scenario에서 예상 매출은 각각 Z원이며, 단 계통 접속 및 출력제어 리스크는 별도 확인이 필요하다.

## 5. 가장 중요한 판단

일사량이 좋은 지역이 항상 좋은 투자지역은 아니다.

태양광 사업성은 아래 순서로 봐야 한다.

1. 발전 잠재량: PVOUT, GHI, GTI
2. 설치 가능성: 토지, 지목, 경사, 음영, 인허가
3. 계통 가능성: 접속 여유, 변전소/선로, 공사비
4. 판매 가능성: SMP, REC, PPA, 자가소비 단가
5. 손실 위험: 출력제어, curtailment, 고장, degradation

우리 사업 아이디어 관점에서는 3번과 5번이 차별점이다. 단순히 “햇빛 많은 곳”을 찾는 서비스는 많다. 더 가치 있는 것은 “햇빛은 좋지만 계통/출력제어 때문에 수익이 깎이는 곳”과 “햇빛은 조금 낮아도 실제 수익성이 더 안정적인 곳”을 구분하는 것이다.
